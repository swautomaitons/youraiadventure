Analysis

Microsoft released three documents this week that form a coherent architectural argument.

A new AI pillar in Zero Trust. An updated enterprise reference architecture. An end-to-end agentic security guide titled "Secure agentic AI."

But the document worth reading is: "From apps to agents: Rearchitecting enterprise work around intent."

The argument is structural: Enterprise software was built around applications with interfaces, permission models, and auditable user actions. Agents operate differently — they act on intent, access multiple systems in sequence, and make decisions without human approval.

Microsoft's response: extend Zero Trust to the AI estate. Five principles applied to agents: 1. Verified identity — every agent is a named principal 2. Least privilege — access scoped to the task, not the system 3. Assume breach — design the blast radius in from the start 4. Explicit verification — every action logged and validated 5. Full auditability — decision trails that survive regulatory review

Cisco, CrowdStrike, and Google Cloud are co-signatories. This is the beginning of an industry standard.
