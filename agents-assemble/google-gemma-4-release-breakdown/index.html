How to Build Offline AI Agents With Gemma 4 — The Step-by-Step Guide for Agent Builders

A hands-on walkthrough from zero to a deployed AI agent. You will set up your environment, define tools, configure memory, build the orchestration loop, test everything, and ship to production.

Featured Image Architecture diagram: User -> LLM -> Tools -> Memory -> Response
TL;DR

Install Python 3.11+ and the openai SDK. Define tools as JSON-schema-annotated functions. Wire up a SQLite-backed memory layer. Run an orchestration loop that calls the LLM, dispatches tool calls, and appends results. Write three focused test cases. Deploy behind FastAPI on a $5/month VPS. Total time: roughly 30 minutes.

What you will need

  • Python 3.11 or higher (python --version)
  • An OpenAI API key (or any OpenAI-compatible endpoint)
  • Basic familiarity with Python functions and dictionaries
  • A terminal and a code editor
Step 1 / 6 · ~5 min

Environment Setup

Create an isolated project directory and virtual environment. This keeps your agent's dependencies separate and prevents version conflicts.

terminal 
mkdir my-first-agent && cd my-first-agent
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

Install the two packages you need: the OpenAI SDK for LLM calls and python-dotenv so your API key stays out of source control.

terminal 
pip install openai python-dotenv

Create a .env file in the project root for your API key:

.env 
OPENAI_API_KEY=sk-your-key-here

Never commit .env to version control. Add it to .gitignore immediately.

Set up the project scaffold -- just three files:

terminal 
touch agent.py tools.py memory.py
echo ".env" >> .gitignore

Step 2 / 6 · ~7 min

Define Your Tools

Tools make an agent more than a chatbot. Each tool is a Python function the LLM can invoke to interact with the outside world. The LLM decides which tool to use based on the JSON schema you provide.

Write the tool functions

tools.py 
import json
import urllib.request
from datetime import datetime


def get_weather(location: str) -> str:
    """Fetch current weather for a given city name."""
    url = f"https://wttr.in/{location}?format=j1"
    try:
        with urllib.request.urlopen(url, timeout=5) as resp:
            data = json.loads(resp.read())
            current = data["current_condition"][0]
            return (
                f"Weather in {location}: "
                f"{current['temp_C']}°C, "
                f"{current['weatherDesc'][0]['value']}"
            )
    except Exception as e:
        return f"Could not fetch weather: {e}"


def calculate(expression: str) -> str:
    """Evaluate a mathematical expression safely."""
    allowed = set("0123456789+-*/.() ")
    if not all(c in allowed for c in expression):
        return "Error: expression contains invalid characters"
    try:
        result = eval(expression)
        return str(result)
    except Exception as e:
        return f"Calculation error: {e}"


def get_current_time() -> str:
    """Return the current date and time."""
    return datetime.now().strftime("%Y-%m-%d %H:%M:%S")

Register the tool schemas

Create the schema definitions that tell the LLM what each tool does. This follows the OpenAI function-calling format.

tools.py (continued) 
TOOL_FUNCTIONS = {
    "get_weather": get_weather,
    "calculate": calculate,
    "get_current_time": get_current_time,
}

TOOL_SCHEMAS = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "Get current weather for a city",
            "parameters": {
                "type": "object",
                "properties": {
                    "location": {
                        "type": "string",
                        "description": "City name, e.g. 'London'"
                    }
                },
                "required": ["location"]
            }
        }
    },
    {
        "type": "function",
        "function": {
            "name": "calculate",
            "description": "Evaluate a math expression",
            "parameters": {
                "type": "object",
                "properties": {
                    "expression": {
                        "type": "string",
                        "description": "Math expression, e.g. '2 + 2'"
                    }
                },
                "required": ["expression"]
            }
        }
    },
    {
        "type": "function",
        "function": {
            "name": "get_current_time",
            "description": "Get current date and time",
            "parameters": {
                "type": "object",
                "properties": {}
            }
        }
    }
]

Step 3 / 6 · ~5 min

Configure Memory

Without memory, your agent forgets everything between sessions. A lightweight SQLite database gives you persistent conversation history with zero infrastructure.

memory.py 
import sqlite3
import json
from pathlib import Path

DB_PATH = Path(__file__).parent / "agent_memory.db"


def _get_conn():
    conn = sqlite3.connect(DB_PATH)
    conn.execute("""
        CREATE TABLE IF NOT EXISTS messages (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            session_id TEXT NOT NULL,
            role TEXT NOT NULL,
            content TEXT,
            tool_calls TEXT,
            tool_call_id TEXT,
            created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
        )
    """)
    return conn


def save_message(session_id: str, message: dict):
    """Persist a single message to the database."""
    conn = _get_conn()
    conn.execute(
        """INSERT INTO messages
           (session_id, role, content, tool_calls, tool_call_id)
           VALUES (?, ?, ?, ?, ?)""",
        (
            session_id,
            message.get("role"),
            message.get("content"),
            json.dumps(message.get("tool_calls"))
                if message.get("tool_calls") else None,
            message.get("tool_call_id"),
        ),
    )
    conn.commit()
    conn.close()


def load_history(session_id: str, limit: int = 50) -> list:
    """Load the most recent messages for a session."""
    conn = _get_conn()
    rows = conn.execute(
        """SELECT role, content, tool_calls, tool_call_id
           FROM messages
           WHERE session_id = ?
           ORDER BY id DESC LIMIT ?""",
        (session_id, limit),
    ).fetchall()
    conn.close()

    messages = []
    for role, content, tool_calls_json, tool_call_id in reversed(rows):
        msg = {"role": role}
        if content:
            msg["content"] = content
        if tool_calls_json and tool_calls_json != "null":
            msg["tool_calls"] = json.loads(tool_calls_json)
        if tool_call_id:
            msg["tool_call_id"] = tool_call_id
        messages.append(msg)
    return messages

For production, swap SQLite for PostgreSQL or Redis. The interface stays the same -- only the storage backend changes.

The limit=50 default keeps your context window manageable. For GPT-4o with 128k context, you can increase this to 200+ messages. For smaller models, keep it around 20-30.

Step 4 / 6 · ~8 min

Build the Orchestration Loop

The orchestration loop is the heart of your agent. It runs a cycle: send messages to the LLM, check if the LLM wants to call a tool, execute the tool, append the result, and repeat until a plain text response comes back.

Step 4 Diagram Flowchart: User Input -> LLM Call -> Tool Call? (yes: execute + loop, no: return response)
agent.py 
import json
import os
from dotenv import load_dotenv
from openai import OpenAI

from tools import TOOL_SCHEMAS, TOOL_FUNCTIONS
from memory import save_message, load_history

load_dotenv()
client = OpenAI()

SYSTEM_PROMPT = """You are a helpful assistant with access to tools.
Use them when the user's request requires real-time data or computation.
Always explain your reasoning before calling a tool."""

MAX_TOOL_ROUNDS = 10


def run_agent(user_input: str, session_id: str = "default"):
    # 1. Load conversation history
    messages = load_history(session_id)

    # 2. Ensure system prompt is first
    if not messages or messages[0].get("role") != "system":
        messages.insert(0, {"role": "system", "content": SYSTEM_PROMPT})

    # 3. Append new user message
    user_msg = {"role": "user", "content": user_input}
    messages.append(user_msg)
    save_message(session_id, user_msg)

    # 4. Orchestration loop
    for _ in range(MAX_TOOL_ROUNDS):
        response = client.chat.completions.create(
            model="gpt-4o",
            messages=messages,
            tools=TOOL_SCHEMAS,
        )
        choice = response.choices[0]
        assistant_msg = choice.message

        msg_dict = {"role": "assistant", "content": assistant_msg.content}
        if assistant_msg.tool_calls:
            msg_dict["tool_calls"] = [
                {
                    "id": tc.id,
                    "type": "function",
                    "function": {
                        "name": tc.function.name,
                        "arguments": tc.function.arguments,
                    },
                }
                for tc in assistant_msg.tool_calls
            ]

        messages.append(msg_dict)
        save_message(session_id, msg_dict)

        # 5. If no tool calls, we have our final answer
        if not assistant_msg.tool_calls:
            return assistant_msg.content

        # 6. Execute each tool call
        for tool_call in assistant_msg.tool_calls:
            fn_name = tool_call.function.name
            fn_args = json.loads(tool_call.function.arguments)

            fn = TOOL_FUNCTIONS.get(fn_name)
            if fn:
                result = fn(**fn_args)
            else:
                result = f"Error: unknown tool '{fn_name}'"

            tool_msg = {
                "role": "tool",
                "content": result,
                "tool_call_id": tool_call.id,
            }
            messages.append(tool_msg)
            save_message(session_id, tool_msg)

    return "Agent reached maximum tool call rounds."


if __name__ == "__main__":
    print("AI Agent Ready. Type 'quit' to exit.\n")
    while True:
        user_input = input("You: ")
        if user_input.lower() in ("quit", "exit"):
            break
        response = run_agent(user_input)
        print(f"\nAgent: {response}\n")

How the loop works

  1. The user message is appended to conversation history.
  2. Full history is sent to the LLM with available tool schemas.
  3. If the LLM returns tool_calls, each is executed and the result appended as a tool message.
  4. The updated history goes back to the LLM.
  5. When the LLM returns plain text (no tool calls), the loop exits.

MAX_TOOL_ROUNDS = 10 is critical. Without it, a misbehaving model could loop forever, burning your API budget.

Step 5 / 6 · ~3 min

Testing Your Agent

Verify your agent handles three key scenarios: a plain question (no tools), a single tool call, and a multi-step reasoning chain requiring multiple tool calls.

test_agent.py 
from agent import run_agent

def test_no_tools():
    """The agent should answer without tools."""
    result = run_agent(
        "What is an AI agent?",
        session_id="test-no-tools"
    )
    assert isinstance(result, str)
    assert len(result) > 20
    print(f"PASS: no-tools -> {result[:80]}...")

def test_single_tool():
    """The agent should use the weather tool."""
    result = run_agent(
        "What's the weather in Tokyo right now?",
        session_id="test-single-tool"
    )
    assert "Tokyo" in result or "tokyo" in result.lower()
    print(f"PASS: single-tool -> {result[:80]}...")

def test_multi_step():
    """The agent should chain multiple tools."""
    result = run_agent(
        "What time is it, and what's 1547 * 23?",
        session_id="test-multi-step"
    )
    assert "35581" in result
    print(f"PASS: multi-step -> {result[:80]}...")

if __name__ == "__main__":
    test_no_tools()
    test_single_tool()
    test_multi_step()
    print("\nAll tests passed.")

Run the tests:

terminal 
python test_agent.py

Step 6 / 6 · ~5 min

Deploy to Production

Wrap your agent in a FastAPI server so it can be called over HTTP. This gives you a REST endpoint accessible from a front-end, mobile app, or another service.

terminal 
pip install fastapi uvicorn
server.py 
from fastapi import FastAPI
from pydantic import BaseModel
from agent import run_agent

app = FastAPI(title="AI Agent API")


class ChatRequest(BaseModel):
    message: str
    session_id: str = "default"


class ChatResponse(BaseModel):
    response: str
    session_id: str


@app.post("/chat", response_model=ChatResponse)
async def chat(req: ChatRequest):
    result = run_agent(req.message, req.session_id)
    return ChatResponse(
        response=result,
        session_id=req.session_id,
    )


@app.get("/health")
async def health():
    return {"status": "ok"}

Start the server locally:

terminal 
uvicorn server:app --reload --port 8000

Test with curl:

terminal 
curl -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "What is 42 * 17?", "session_id": "test"}'

Deploy to a VPS

A $5/month VPS (DigitalOcean, Hetzner, Railway) is more than enough. Minimal deployment script:

deploy.sh 
#!/bin/bash
git clone your-repo-url my-agent && cd my-agent
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
echo "OPENAI_API_KEY=sk-your-key" > .env
pip install gunicorn
gunicorn server:app -w 2 -k uvicorn.workers.UvicornWorker \
  --bind 0.0.0.0:8000

Add rate limiting and authentication before exposing to the public internet. slowapi and a simple API key middleware will cover the basics.

Key Takeaways

An AI agent is a loop: call the LLM, execute requested tools, feed results back, repeat until you get a text response.

Tools are the superpower. Define them as plain Python functions with JSON schemas, and the LLM figures out when to use them.

Memory is non-negotiable for real use. Even a SQLite file gives persistent, session-aware conversations.

Always cap tool-calling rounds. A runaway loop drains your API budget fast.

Test no-tools, single-tool, and multi-tool chains. Three tests cover the critical paths.

FastAPI + Gunicorn = localhost to production in under 5 minutes.
